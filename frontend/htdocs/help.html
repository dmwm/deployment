Help and Support

Planned CMSWEB downtimes are listed in the scheduled interventions page. These interventions and other CMSWEB important issues and changes are reported in the CERN computing announcements forum.

To authenticate against CMSWEB services, you must be properly registered with CMS. The trouble-shooting page helps you to diagnose certificate authentication problems. Please also check your certificate is properly mapped to your account in SiteDB.

If you constantly having your browser complaining this site is not trusted (and need to confirm an exception), it means you don't have the CERN Grid CA certificates properly installed. Please follow the instructions here. If you had it already installed, make sure you have marked all the "Trust this CA to..." check boxes as the instructions tell.

Some services may additionally require you to have specific roles in SiteDB to allow you to access some functions or features. Look for them in the respective service help pages and follow corresponding instructions. If you don't know where to request or find information about, ask the cms-service-webtools CERN e-group.

For CMSWEB services API documentation, see apidocs.

For reporting bugs or any other CMSWEB problems, please open a ticket and assign it to "CMS Web Tools" Support Unit.
